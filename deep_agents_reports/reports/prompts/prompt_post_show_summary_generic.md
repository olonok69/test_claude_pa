## CYPHER QUERIES

Execute the following queries to gather the required data:

### Query 1: Registered Visitors Breakdown

```cypher
// Registered visitors with/without recommendations
MATCH (v:Visitor_this_year)
WHERE v.show = '[SHOW_CODE]'
WITH 
  count(v) AS total_registered,
  count(CASE WHEN EXISTS((v)-[:IS_RECOMMENDED]->()) THEN 1 END) AS with_recs,
  count(CASE WHEN NOT EXISTS((v)-[:IS_RECOMMENDED]->()) THEN 1 END) AS without_recs
RETURN 'Registered' AS stage, with_recs, without_recs, total_registered AS total
```

### Query 2: Attended Show Breakdown

```cypher
// Visitors who attended the show (venue entry via badge scan)
MATCH (v:Visitor_this_year)-[:registered_to_show]->(show:Show_this_year)
WHERE v.show = '[SHOW_CODE]'
WITH 
  count(v) AS total_attended_show,
  count(CASE WHEN EXISTS((v)-[:IS_RECOMMENDED]->()) THEN 1 END) AS with_recs,
  count(CASE WHEN NOT EXISTS((v)-[:IS_RECOMMENDED]->()) THEN 1 END) AS without_recs
RETURN 'Attended show' AS stage, with_recs, without_recs, total_attended_show AS total
```

### Query 3: Attended Any Sessions Breakdown

```cypher
// Visitors who attended at least 1 session
MATCH (v:Visitor_this_year)-[:assisted_session_this_year]->(s:Sessions_this_year)
WHERE v.show = '[SHOW_CODE]' AND s.show = '[SHOW_CODE]'
WITH v, count(s) AS sessions_attended
WHERE sessions_attended >= 1
WITH 
  count(v) AS total_attended_sessions,
  count(CASE WHEN EXISTS((v)-[:IS_RECOMMENDED]->()) THEN 1 END) AS with_recs,
  count(CASE WHEN NOT EXISTS((v)-[:IS_RECOMMENDED]->()) THEN 1 END) AS without_recs
RETURN 'Attended any sessions' AS stage, with_recs, without_recs, total_attended_sessions AS total
```

### Query 4: Attended Recommended Sessions

```cypher
// Visitors who attended at least 1 session that was recommended to them
MATCH (v:Visitor_this_year)-[:IS_RECOMMENDED]->(s:Sessions_this_year)
WHERE v.show = '[SHOW_CODE]' AND s.show = '[SHOW_CODE]'
AND EXISTS((v)-[:assisted_session_this_year]->(s))
WITH DISTINCT v
RETURN count(v) AS attended_recommended_sessions
```

### Query 5: Summary Statistics

```cypher
// Get summary statistics for the report
MATCH (v:Visitor_this_year)
WHERE v.show = '[SHOW_CODE]'
OPTIONAL MATCH (v)-[rec:IS_RECOMMENDED]->(s:Sessions_this_year)
WHERE s.show = '[SHOW_CODE]'
WITH v, count(rec) AS recs_count
RETURN 
  count(v) AS total_visitors,
  sum(recs_count) AS total_recommendations,
  avg(CASE WHEN recs_count > 0 THEN recs_count END) AS avg_recs_per_visitor_with_recs
```

## KEY METRICS TO CALCULATE

After gathering the raw data, calculate and present:

### Coverage & Reach Metrics

| Metric | Formula |
|--------|---------|
| **Recommendation Coverage** | Visitors with recs / Total registered × 100% |
| **Show Attendance Rate** | Attended show / Total registered × 100% |
| **Session Engagement Rate** | Attended any sessions / Total registered × 100% |

### Recommendation Effectiveness Metrics

| Metric | Formula |
|--------|---------|
| **Visitor-Level Hit Rate** | Attended recommended sessions / Visitors with recs × 100% |
| **Session Engagement (With Recs)** | Attended sessions (with recs) / Visitors with recs × 100% |
| **Session Engagement (No Recs)** | Attended sessions (no recs) / Visitors without recs × 100% |
| **Engagement Difference** | Engagement with recs − Engagement without recs (in percentage points) |
| **Engagement Lift** | (Engagement with recs − Engagement without recs) / Engagement without recs × 100% |

> **Note for stakeholders:** Always present BOTH metrics:
> - **Engagement Difference (pp)**: The absolute percentage point difference — easy to understand
> - **Engagement Lift (%)**: The relative increase from baseline — shows proportional impact
>
> Example: "81.3% vs 60.2% = +21.1 pp absolute difference, which is a +35.1% relative lift"

### Funnel Conversion Metrics

| Transition | Formula |
|------------|---------|
| **Registered → Attended Show** | Attended show / Registered × 100% |
| **Attended Show → Attended Sessions** | Attended sessions / Attended show × 100% |
| **With Recs → Attended Recommended** | Attended recommended / With recs × 100% |

## DATABASE REQUIREMENTS

This report requires the following Neo4j relationships:

| Relationship | Description | Required |
|--------------|-------------|:--------:|
| `IS_RECOMMENDED` | Pre-show recommendations (visitor → session) | ✅ |
| `registered_to_show` | Venue entry badge scans (visitor → show) | ✅ |
| `assisted_session_this_year` | Session attendance badge scans (visitor → session) | ✅ |

**Verification Query:**
```cypher
// Verify all required data is present
MATCH (v:Visitor_this_year) WHERE v.show = '[SHOW_CODE]'
OPTIONAL MATCH (v)-[r1:IS_RECOMMENDED]->()
OPTIONAL MATCH (v)-[r2:registered_to_show]->()
OPTIONAL MATCH (v)-[r3:assisted_session_this_year]->()
RETURN 
  count(DISTINCT v) AS visitors,
  count(DISTINCT r1) AS recommendations,
  count(DISTINCT r2) AS show_registrations,
  count(DISTINCT r3) AS session_attendances
```
